Repo-local agent harness toolkit

Harness Hub

让 agent 在不同仓库里使用同一套可重复、可验证、可交接的工作方式。它负责技能安装、minimal harness 初始化、验证和托管文件生命周期安全。

首次理解

先看状态

check 和 analyze 是只读入口，用来判断目标仓库是否已经准备好。

再初始化 harness

init-harness 只在你明确确认后创建 minimal 根级 harness 文件。

只装技能时分开做

install 只安装标准 skill set，不创建根级 harness 文件。

安全起步

  1. 在目标仓库外先运行 dry-run，检查计划写入的文件。
  2. 确认目标仓库的现有规则、锁文件和本地 state 不会被误覆盖。
  3. 只有看懂计划后再把命令切换到 --yes。
npx @jasonwen/harness-hub init-harness D:\path\to\target --target standard --dry-run --json

Harness Hub 不会自动创建定时任务、webhook、commit、push、全局 skill 安装或远程服务改动，除非某条命令明确声明这类副作用。
