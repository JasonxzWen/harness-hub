# Source Blog Checklist

- [ ] Canonical source URL, title, author, publish date, and access date captured.
- [ ] Access limitations recorded.
- [ ] Reader job is explicit: first-minute answer, source structure, and project implication.
- [ ] Source headings and order preserved.
- [ ] Media ledger includes URL, type, alt text, caption, and placement.
- [ ] Important links and named references are preserved.
- [ ] Tables, footnotes, and glossary entries are represented.
- [ ] Reader-facing article layer appears before evidence ledger, generated report, or implementation details.
- [ ] Any early visual is limited to one helpful tree, outline, diagram, or compact comparison.
- [ ] Tables are used only for comparable rows; paragraph-length table cells were rewritten as prose.
- [ ] Copyright boundary appears in the public post.
- [ ] Effective-interact summary or visualization layer exists and is secondary.
- [ ] Project-iteration review says already changed / effective / redundant risk / next optimization / not now.
- [ ] 60-second reader test passes: a reader can say what the source is, how it is organized, what changed locally, and whether it was worth it.
- [ ] Rendered screenshot confirms the blog is readable.
- [ ] `validate-source-blog.mjs <post-dir>` passes.
- [ ] Insight site validation passes.
